# 08｜Hermes AI 與 Telegram 串接導入流程

## 1. 目標

把 `出團訊息chat box md` 的 Markdown 資料庫變成 Hermes AI 可以讀取的智慧資料庫，並透過 Telegram Bot 使用。

完成後，使用者可以在 Telegram 直接輸入：

```text
幫我產出泰國曼谷當地注意事項
```

Hermes AI 應自動：

```text
讀取 03 判斷國家 / 城市
讀取 02 抓泰國國家資料
讀取 07 抓曼谷城市補充
讀取 04 套旅客公告模板
輸出可直接貼給旅客群組的公告
```

---

## 2. 建議架構

建議使用：

```text
GitHub = 主資料庫
Hermes AI = 讀取資料庫、理解需求、產出回覆
Telegram Bot = 使用入口
```

不建議讓 Hermes AI 成為唯一資料庫，因為 GitHub 比較容易追蹤修改紀錄、回復舊版本與備份。

## 4. Hermes AI 設定方式

Hermes AI 需要知道三件事：

1. 資料庫在哪裡。
2. 收到 Telegram 訊息時要讀哪些 MD。
3. 輸出格式要照哪份模板。

### 4.1 Hermes 系統提示詞建議

可把以下內容放進 Hermes AI 的系統提示詞或 Agent instructions：

```text
你是 Yoyo 的出團訊息助理，負責依照 Markdown 資料庫產出可直接貼到旅客 LINE / Telegram / WhatsApp 群組的公告。

你必須優先讀取出團訊息 chat box md 資料夾中的資料：

01_班機登機與下機提醒模板.md
02_東南亞國家基本資料庫.md
03_國家判斷與產出規則.md
04_當地注意事項懶人包公版模板.md
05_飯店入住退房與明日行程提醒模板.md
06_Telegram_Bot_指令與輸入格式.md
07_城市與區域補充資料庫.md
09_必買必吃必拍旅遊推薦資料庫.md
13_機場說明會公版模板.md

當使用者要求產出國家 / 城市 / 當地注意事項時：
1. 先讀取 03 判斷國家、城市、區域或雙國行程。
2. 再讀取 02 抓國家基本資料。
3. 若有城市、區域或景點，讀取 07 補充城市區域提醒。
4. 最後讀取 04 套用當地注意事項懶人包格式。

當使用者要求班機、登機、下機、行李轉盤提醒時，使用 01。

當使用者要求飯店入住、退房、明日行程、晨喚、早餐、集合提醒時，使用 05。

當使用者要求必買、必吃、必拍、伴手禮、推薦美食或推薦景點時，使用 09。

當使用者要求機場說明會、機場集合說明、掛行李前說明、出發當天說明、領隊口說稿或現場提詞卡時，使用 13。

缺少資料時保留【欄位】，不要自行亂補。

產出旅客公告時，每次只輸出一份完整最終版，不要先輸出一半或草稿，再重新產出完整版本。若內容太長，優先壓縮文字並保留完整結構；必要時才分成「第 1 段／第 2 段」，且第 2 段只接續未完成內容，不要從頭重來。

涉及簽證、入境表單、海關、菸酒、電子菸、匯率、天氣、航班等容易變動資訊時，需提醒以官方公告、航空公司與旅行社通知為準。

輸出必須使用繁體中文，語氣自然、清楚、親切，像專業領隊提醒旅客，可直接貼到旅客群組。
```

## 5. Telegram Bot 設定流程

### 5.1 建立 Telegram Bot

1. 到 Telegram 找 `@BotFather`。
2. 輸入 `/newbot`。
3. 設定 bot 名稱。
4. 取得 bot token。

### 5.2 連接 Hermes AI

在 Hermes AI 裡找到 Telegram integration / Telegram gateway 設定，填入：

```text
Telegram Bot Token
允許使用者 ID / 群組 ID
Hermes Agent ID 或對應的 Agent
```

### 5.3 測試訊息

在 Telegram 對 bot 輸入：

```text
幫我產出泰國曼谷當地注意事項
```

理想輸出應該是：

```text
各位貴賓您好，以下是【泰國】當地注意事項懶人包：

⭕️ 時差
泰國比台灣慢 1 小時。

...
```

## 6. 更新資料流程

不建議讓 Hermes AI 直接自動修改 MD。建議使用「先比對、再確認、後更新」流程。

### 建議流程

```text
使用者：幫我查泰國 TDAC 最新規定，和 02 比對，不要直接修改。
↓
Hermes AI：查官方或指定來源。
↓
Hermes AI：列出目前 MD 與最新資料差異。
↓
使用者：確認更新。
↓
Hermes AI：更新 GitHub / MD 檔案。
```

### 更新規則

- 官方來源優先。
- 使用者提供的旅行社最新通知優先。
- 更新前先列出差異。
- 不可未經確認直接修改簽證、入境、海關、菸酒、電子菸等高風險欄位。

## 8. 成功標準

如果 Bot 能做到以下幾點，就代表第一版成功：

- 能判斷國家與城市。
- 能讀取正確國家資料，不混用規定。
- 能補入城市與區域提醒。
- 能依照 `04` 的公告格式輸出。
- 能依照 `13` 產出機場說明會完整逐字稿或現場提詞卡。
- 缺資料時會保留【欄位】。
- 不會自行亂補最新法規。
- 輸出內容可直接貼給旅客群組。
