# 06｜Telegram Bot 指令與輸入格式

## 1. 這份資料的用途

這份資料提供 Telegram Bot / AI Chatbox 判斷使用者需求、讀取對應 Markdown 資料、產出旅客群組公告時使用。

使用者不需要輸入複雜指令，可以用自然語言輸入。Bot 的任務是先判斷情境，再決定要讀取哪幾份資料。

---

## 2. Bot 核心工作流程

AI 收到使用者訊息後，依照以下順序處理：

1. 判斷使用者想產出哪一種訊息。
2. 判斷是否有指定國家、城市、區域或雙國行程。
3. 讀取對應 Markdown 資料。
4. 把使用者提供的資訊填入模板。
5. 缺少資料時保留【欄位】，不要自行亂補。
6. 產出可直接貼到 LINE / Telegram / WhatsApp 群組的公告版本。

## 4. 最重要流程：國家當地注意事項

這是本系統最核心的功能。當使用者提到以下需求時，Bot 應優先判斷為「國家當地注意事項」：

- 當地注意事項
- 行前提醒
- 旅遊注意事項
- 入境提醒
- 懶人包
- 國家提醒
- 出團提醒
- 注意什麼
- 禁帶什麼
- 插座、時差、換錢、入境表單、菸酒限制

### 4.1 國家注意事項讀取流程

```text
使用者輸入
↓
讀取 03_國家判斷與產出規則.md
↓
判斷國家 / 城市 / 區域 / 雙國行程
↓
讀取 02_東南亞國家基本資料庫.md
↓
若有城市 / 區域 / 景點，讀取 07_城市與區域補充資料庫.md
↓
讀取 04_當地注意事項懶人包公版模板.md
↓
產出旅客群組公告
```

### 4.2 最簡單輸入格式

使用者只要輸入：

```text
幫我產出【國家／城市】注意事項
```

範例：

```text
幫我產出泰國曼谷注意事項
```

```text
幫我產出越南中越行前提醒
```

```text
幫我做新馬 5 日注意事項懶人包
```

```text
幫我產出印尼峇里島當地注意事項
```

### 4.3 推薦輸入格式

若使用者想讓內容更準，可使用：

```text
幫我產出【國家／城市】當地注意事項。
行程天數：【天數】
城市：【城市一、城市二】
天氣：【溫度／降雨機率，沒有可留空】
特殊行程：【寺廟／清真寺／海島／山區／玩水／巴拿山】
版本：【簡短版／完整版／長輩友善版／群組公告版】
```

範例：

```text
幫我產出越南中越當地注意事項。
行程天數：5 天
城市：峴港、會安、巴拿山
特殊行程：巴拿山、會安古城
版本：群組公告版
```

Bot 應加入：

- 越南時差慢台灣 1 小時
- 越南盾與換匯提醒
- 越南簽證依區域不同
- 中越城市與區域提醒需從 `07_城市與區域補充資料庫.md` 補充
- 歐規兩孔圓腳，需轉接頭
- 電子菸嚴禁
- 巴拿山薄外套與雨具
- 出境安檢可能需脫鞋

### 4.4 國家注意事項必要輸出欄位

國家注意事項至少應包含：

- 時差
- 天氣與溫度
- 穿著與用品
- 宗教與服裝
- 電壓與插頭
- 換錢
- 網路與通訊
- 盥洗用具
- 個人藥品
- 入境資料
- 入境與禁帶提醒
- 護照提醒
- 小費與用水
- 其他國家別提醒

若使用者要求「簡短版」，至少保留：

- 時差
- 天氣與穿著
- 插座
- 換錢
- 入境資料
- 禁帶物品
- 服裝或特殊提醒

### 4.5 國家注意事項標準輸出樣式

Bot 產出國家注意事項時，應以旅客公告口吻自然寫成完整句，不要只做欄位列表。

標準順序如下：

```text
各位貴賓您好，以下是【國家】當地注意事項懶人包：

⭕️ 時差
⭕️ 天氣與溫度
⭕️ 穿著與用品
⭕️ 宗教與服裝
⭕️ 電壓與插頭
⭕️ 換錢
⭕️ 網路與通訊
⭕️ 飯店盥洗用品
⭕️ 個人藥品
⭕️ 入境資料
⭕️ 入境與禁帶提醒
⭕️ 護照提醒
⭕️ 小費與用水
⭕️ 其他提醒

請協助通知同行親友，謝謝大家。
```

若某國有特殊宗教服裝規定，標題可改成更明確的名稱：

- 泰國：佛寺穿著
- 馬來西亞：清真寺穿著
- 菲律賓：教堂與宗教場所

若某個欄位不適用，可用一般提醒替代，不要硬寫錯誤資訊。

### 4.6 國家資料不可混用規則

Bot 產出國家注意事項時，必須遵守：

- 不可把泰國 TDAC 寫到越南。
- 不可把馬來西亞 MDAC 寫到新加坡。
- 不可把新加坡口香糖禁止寫到所有國家。
- 不可把新加坡、馬來西亞菸品全面課稅寫到所有國家。
- 不可把巴拿山薄外套提醒寫到越南全境，除非使用者提到中越、峴港、會安、巴拿山。
- 不可把峇里島無時差套用到雅加達。
- 不可把英規三方扁腳套用到泰國或菲律賓。
- 不可在簽證、入境、菸酒、電子菸、匯率、天氣資料不足時自行補最新法規。

## 8. 版本與語氣控制

使用者可以在輸入最後加上版本需求。

| 版本 | Bot 輸出方式 |
|---|---|
| 簡短版 | 保留重點，適合快速發群組 |
| 完整版 | 欄位完整，適合行前詳細公告 |
| 長輩友善版 | 用字更白話，提醒更明確 |
| 群組公告版 | 自然、親切，可直接貼 LINE / Telegram |
| 條列版 | 用項目符號整理 |
| 檢查表版 | 適合領隊自己核對 |

若使用者沒有指定，預設使用「群組公告版」。

## 9. 缺少資料時的處理

Bot 不可亂補使用者沒有提供、資料庫也沒有的資訊。

若缺少資料，使用以下方式處理：

```text
【欄位名稱】
```

例如：

```text
溫度：【溫度範圍】
降雨機率：【降雨機率】
登機門：【登機門】
行李轉盤：【行李轉盤號碼】
```

若是容易變動的資訊，Bot 應加入：

```text
實際規定請以出發前官方公告、航空公司與旅行社通知為準。
```

## 10. 輸出格式規則

Bot 輸出旅客公告時，應遵守：

- 使用繁體中文。
- 語氣自然、清楚、親切，像專業領隊提醒旅客。
- 不要過度官方。
- 不要恐嚇旅客。
- 不要輸出內部判斷流程，除非使用者要求。
- 可以直接貼到 LINE / Telegram / WhatsApp 群組。
- 若是旅客公告，開頭可使用「各位貴賓您好」。
- 結尾可使用「請協助通知同行親友，謝謝大家」。

## 12. Bot 判斷優先順序

若使用者一句話同時提到多種情境，Bot 依以下優先順序判斷：

1. 若有明確國家 / 城市 + 注意事項 / 行前提醒，優先走國家當地注意事項流程。
2. 若有航班、登機、下機、行李轉盤，走班機提醒流程。
3. 若有飯店、入住、退房、明日行程、集合時間，走飯店與行程提醒流程。
4. 若有必買、必吃、必拍、伴手禮、推薦景點，走旅遊推薦資料流程。
5. 若無法判斷，先產出可填寫版本，並保留【欄位】。

若同時需要多種公告，例如「明天泰國出發、幫我做登機提醒和當地注意事項」，Bot 應分成兩段產出：

```text
一、登機提醒
二、泰國當地注意事項
```
