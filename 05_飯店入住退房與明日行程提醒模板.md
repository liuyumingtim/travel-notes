# 05｜飯店入住退房與明日行程提醒模板

## 1. 這份資料的用途

這份資料是提供 AI / Telegram Chatbot 產出「飯店入住提醒」、「退房提醒」與「明日行程提醒」時使用。

主要用於：

- 飯店入住後群組提醒
- 飯店設備檢查提醒
- 隔日集合時間通知
- 退房前物品檢查提醒
- 明日行程與攜帶物品提醒
- 領隊對旅客的 LINE / Telegram / WhatsApp 群組公告

---

## 2. 產出邏輯

當使用者要求產出飯店相關提醒時，AI 應先判斷使用者需要哪一種版本：

1. 飯店入住提醒
2. 飯店退房提醒
3. 明日行程提醒
4. 飯店入住＋明日行程整合版
5. 退房＋明日行程整合版

### 2.1 單一需求優先規則

若使用者已明確指定單一需求，AI 必須只輸出該需求，不可把其他提醒段落一起輸出：

- 使用者說「入住提醒」、「飯店入住提醒」、「今晚入住提醒」、「給我飯店提醒」且沒有提到退房、明日行程或集合時間時，只輸出「飯店入住提醒」，不要輸出退房提醒。
- 使用者說「退房提醒」、「明天退房提醒」、「離開飯店提醒」、「帶大行李提醒」時，只輸出「飯店退房提醒」，不要輸出入住檢查提醒。
- 使用者說「明日行程提醒」、「明天集合」、「晨喚早餐集合」時，只輸出「明日行程提醒」，不要自動加入入住或退房段落。
- 使用者只說「明日行程」、「明天行程」、「今天行程」、「行程」並提供景點或地點，但沒有提到提醒、公告、晨喚、早餐、集合、行李、攜帶物品或穿著建議時，只輸出「純行程清單」，不要加入攜帶物品、穿著建議、溫馨提醒或未提供的欄位。
- 只有使用者明確說「整合版」、「入住加明日行程」、「退房加明日行程」，或同時提供飯店、行程、集合與行李資訊時，才可輸出整合版。
- 若使用者只說「飯店提醒」但沒有指定入住或退房，預設產出「飯店入住提醒」。若不確定，可先反問「你要入住提醒、退房提醒，還是明日行程提醒？」

若使用者沒有特別指定，且提供了飯店名稱、集合時間、集合地點與明日行程，AI 可產出「飯店入住＋明日行程整合版」。

## 3. 可替換欄位

| 欄位 | 說明 |
|---|---|
| 【飯店名稱】 | 今晚入住飯店名稱 |
| 【日期】 | 明日日期或行程日期 |
| 【國家／城市】 | 行程所在國家或城市 |
| 【景點一】 | 明日第一個景點 |
| 【景點二】 | 明日第二個景點 |
| 【景點三】 | 明日第三個景點 |
| 【景點四】 | 明日第四個景點 |
| 【晨喚時間】 | 飯店 Morning Call 時間 |
| 【早餐時間】 | 早餐開始或集合用餐時間 |
| 【集合時間】 | 旅客集合時間 |
| 【集合地點】 | 飯店大廳、餐廳、遊覽車旁等 |

| 【鞋子／衣著／防曬／雨具提醒】 | 依明日行程補充穿著建議 |

## 4. 公用模板 05｜飯店入住提醒與退房提醒

### 4.1 飯店入住提醒模板

#### 使用時機

當旅客完成飯店入住，或領隊需要提醒旅客檢查房間設備、確認隔日集合資訊時使用。

#### 模板內容

各位貴賓您好，今晚入住【飯店名稱】。

進房後請協助確認：

- 熱水、礦泉水、馬桶、排水是否正常。
- 電話、電視、電燈、冷氣、吹風機是否能使用。

若有問題，請盡快跟我說，我會協助處理。

### 4.2 飯店退房提醒模板

#### 使用時機

當旅客隔天要退房、離開飯店、需要提醒檢查個人物品與集合資訊時使用。

#### 模板內容

各位貴賓您好，明天退房前請再次檢查個人物品。

離開飯店前，請檢查：

- 護照
- 錢包
- 手機
- 充電器
- 行動電源
- 衣櫃
- 抽屜
- 浴室

明日集合時間：【日期／時間】  
集合地點：【飯店集合位置】  
行李安排：【帶大行李／不帶大行李】  

請大家依集合時間準時抵達，貴重物品請隨身保管。謝謝大家。

## 5. 純行程清單與明日行程提醒模板

### 5.1 純行程清單模板

#### 使用時機

當使用者只是要「明日行程」、「今天行程」、「行程順序」或提供一串景點時使用。這不是旅客公告，不要自動加入提醒段落。

#### 模板內容

各位貴賓您好，明日行程如下：

1. 【景點一】
2. 【景點二】
3. 【景點三】
4. 【景點四】

#### 輸出限制

- 不要加入晨喚時間、早餐時間、集合時間、集合地點，除非使用者有提供。
- 不要加入攜帶物品、穿著建議、溫馨提醒，除非使用者明確要求「提醒」、「公告」、「群組版」或「要注意什麼」。
- 不要用【欄位】補未提供資訊；純行程清單只列使用者提供的景點或地點。

### 5.2 明日行程提醒模板

#### 使用時機

當領隊需要在晚上提醒旅客隔日行程、晨喚、早餐、集合時間與攜帶物品時使用。

#### 模板內容

日期：【日期】  
地點：【國家／城市】

## 明日行程

1. 【景點一】
2. 【景點二】
3. 【景點三】
4. 【景點四】

晨喚時間：【晨喚時間】  
早餐時間：【早餐時間】  
集合時間：【集合時間】  
集合地點：【集合地點】  

## 攜帶物品

護照、手機、錢包、雨具、外套、水、個人藥品。

## 穿著建議

【鞋子／衣著／防曬／雨具提醒】

## 溫馨提醒

請依集合時間準時抵達，貴重物品請隨身保管。  
若有任何身體不適或特殊需求，請提前告訴我。

## 7. AI 回答規則

當使用者詢問「飯店提醒」、「入住提醒」、「退房提醒」、「明日行程」、「明天集合時間」、「晨喚早餐集合」相關內容時，AI 應遵守以下規則：

- 優先使用本文件模板產出。
- 使用者明確說「入住提醒」時，只產出入住提醒，不要附退房提醒。
- 使用者明確說「退房提醒」時，只產出退房提醒，不要附入住提醒。
- 使用者只說「飯店提醒」且沒有提到退房、明日行程或集合資訊時，預設產出入住檢查提醒。
- 使用者只說「明日行程」、「明天行程」、「行程」並列出景點時，只產出純行程清單，不要加入攜帶物品、穿著建議、溫馨提醒或未提供欄位。
- 若使用者只提供飯店名稱，產出入住檢查提醒。
- 若使用者提供景點、晨喚、早餐、集合時間，或明確要求提醒公告，才產出明日行程提醒。
- 若使用者同時提供飯店與明日行程資訊，優先產出整合版。
- 提醒公告中沒有提供的資訊可使用【欄位】保留，不要亂補；純行程清單不可保留未提供欄位。
- 景點數量可依使用者提供調整，不一定限制 4 個。
- 攜帶物品可以依景點特性補充，例如寺廟、玩水、山區、雨天、高海拔，但只在使用者要求提醒、公告、群組版或注意事項時加入。
- 語氣要自然、清楚、親切，適合直接貼到旅客群組。
- 若使用者提供特殊狀況，例如隔天要退房、要帶大行李、會玩水、要上山，需把提醒加入公告。
